## Dev environment tips
- Follow coding guidelines/pattern from eslint and prettier. This means `pnpm precheck` when ran
- Code change should not throw an error when running `pnpm start` and `pnpm build`
- Agent should ALWAYS ask the user if any changes requires installing additional package.

## MDX instructions
- Check if there are any typos or grammar issues in the .mdx files. Suggest changes if any.
